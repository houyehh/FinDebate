# Bull vs Bear Arena Demo Voiceover Script

Target length: about 2 minutes 25 seconds.

Use this as Mandarin voiceover. The video frames include English captions, so judges can still follow the demo if the audio is not in English.

## 0:00-0:12

大家好，這是 Bull vs Bear Arena，一個用來訓練投資判斷力的本機應用。

它不是下單工具，也不是叫 AI 幫你直接買賣；它的核心是讓你先看完多空辯論，自己先做盲判，然後才揭曉裁判評分。

## 0:12-0:25

我想解決的問題是：投資人很容易被 AI 結論或分數錨定。

所以這個產品把流程反過來：先讓多頭和空頭各自提出證據，再要求使用者先站邊，最後才公布裁判分數。

## 0:25-0:40

首頁可以輸入任何 yfinance 支援的 ticker，例如 NVDA、2330.TW 或 BTC-USD。

後端會驗證標的，回傳公司名稱、目前價格、幣別，以及最近三十天價格走勢。

## 0:40-1:02

按下開始辯論後，系統會產生固定兩輪辯論。

第一輪是多頭與空頭的開場陳述，各有三個論點，每個論點都必須包含主張、證據、來源名稱與來源連結。

## 1:02-1:20

第二輪是交互反駁。每一方會針對對方的兩條論點提出反駁，而且反駁也必須附上來源，不能只靠語氣或空泛判斷。

## 1:20-1:38

最重要的是這裡的盲判流程。

裁判其實已經在背景完成評分，但畫面不先顯示分數。使用者必須先選擇看多、看空或中立觀望，並設定一到五分的信心度。

## 1:38-1:55

送出站邊後，系統才揭曉裁判評分。裁判會從證據具體度、來源品質、邏輯三個面向評分，並標記無法查證或疑似幻覺的論點。

## 1:55-2:13

每一次判斷都會存進 SQLite，戰績頁會在一日、七日和三十日後用真實市場價格回測。

這裡也會顯示勝率、信心校準度，以及你跟裁判同邊或不同邊時的表現差異。

## 2:13-2:28

這個專案是用 Codex 協作完成的，包括 FastAPI 後端、React 前端、SQLite 回測、OpenAI 結構化輸出、BYOK 設定頁，以及 Demo Mode。

正式模式使用 GPT-5.6 產生多頭、空頭與裁判；Demo Mode 則讓評審在 API 額度不足時也能完整測試流程。

## 2:28-2:38

Bull vs Bear Arena 的重點不是預測市場，而是訓練使用者把自己的投資判斷記錄下來，並接受後續真實價格的驗證。
